📚 Course Content — 8 Modules (Seed Data Required)
Seed all 8 modules with full content. Each module must have a minimum of 15 hand-written questions across 4 difficulty tiers (beginner/intermediate/advanced/expert) in the database at launch.
Module 1 — Market Structure
Concepts: Higher Highs/Lows, Lower Highs/Lows, Break of Structure (BOS), Change of Character (CHoCH), Internal vs External Structure, Swing High/Low identification rules, Multi-timeframe structure alignment (Daily → 1H → 15m → 5m).
Chart Scenarios to seed:

Nifty Daily Jan 2024 rally (clean uptrend structure)
BankNifty 15m Oct 2023 (structure break mid-session)
Nifty 1H Feb 2024 (CHoCH before reversal)

Advanced test scenarios: Identify CHoCH on 5m against 1H trend. Multi-timeframe conflict resolution. Distinguish genuine BOS from liquidity sweep that looks like BOS.

Module 2 — Liquidity Theory
Concepts: Buyside/Sellside Liquidity, Equal Highs/Lows, Stop Hunt mechanics, Round number liquidity, Previous Day High/Low as liquidity pools, Liquidity voids, Inducement vs genuine liquidity.
Advanced test scenarios: Identify inducement vs real liquidity pool. Triple equal highs — which one gets swept first? Opening gap liquidity scenarios. BankNifty pre-event liquidity stacks.

Module 3 — Order Blocks
Concepts: Bullish/Bearish OB identification, OB validation rules (BOS requirement), OB mitigation, Breaker Blocks (invalidated OB that flips), Propulsion Blocks, OB refinement on LTF, OB stacking, Volume at OB.
Advanced test scenarios: Distinguish valid OB from random candle. Breaker block identification after OB is violated. Refined OB entry on 5m inside 1H OB. Multiple OBs — which one to prioritize.

Module 4 — Fair Value Gaps & Imbalances
Concepts: Bullish/Bearish FVG identification, Inverse FVG (IFVG), FVG fill probability rules, Partial vs full fills, FVG as support/resistance, Consequent Encroachment (50% of FVG), Balanced Price Range (BPR = overlapping bull+bear FVG), FVG + OB confluence (PD Array Stack).
Advanced test scenarios: Identify BPR on BankNifty 5m. IFVG scenario — when does a filled FVG become resistance? FVG stacking — multiple unfilled FVGs in same direction.

Module 5 — Premium & Discount + Market Maker Model
Concepts: Equilibrium (50% of range), Premium/Discount zones, Optimal Trade Entry (OTE) using Fibonacci 61.8–78.6%, 4-Phase Market Maker Model (Accumulation → Manipulation → Distribution/Expansion → Reversal), Opening Range Manipulation, Daily profile patterns.
Advanced test scenarios: OTE zone calculation on live scenario. Identify which phase the market is in mid-session. Distinguish accumulation consolidation from distribution. Opening manipulation direction identification.

Module 6 — Multi-Timeframe Analysis (MTFA)
Concepts: Top-down analysis framework (Weekly → Daily → 4H → 1H → 15m → 5m), HTF bias establishment, LTF entry triggers, Conflicting timeframe resolution, When to trade with vs against HTF, Timeframe alignment for high-probability entries.
Advanced test scenarios: 5 charts at different timeframes — identify the correct entry TF. HTF bearish vs LTF bullish setup — what to do. Timeframe alignment scoring system.

Module 7 — Trade Management & Risk
Concepts: SL placement rules (beyond sweep wick), Partial profit taking, Breakeven strategy, Trailing SL mechanics (structural trail vs candle trail), Position sizing (1% risk rule), RR calculation, Scaling in/out, Managing trades through news events.
Advanced test scenarios: Given entry/SL/target — calculate exact lot size for 1% risk. Trade management decision tree (what to do when T1 hit, price consolidates, then continues). SL placement for different setup types.

Module 8 — Full System Integration + Chop Detection
Concepts: The 8-point pre-trade checklist, Chop market identification, Range-bound vs trending day classification, High-probability vs low-probability setup scoring, Journal analysis patterns, Pre-market routine (GIFT Nifty, PDH/PDL, economic calendar), The psychology of skipping trades.
Advanced test scenarios: Full day chart — find ALL valid setups (not just one). Chop day identification from first 30 minutes. Complete pre-trade checklist scoring on 3 different setups — one passes, two fail, identify which.

🤖 AI Question Generation System
Backend Endpoint: POST /api/ai/generate-questions
When a user is about to take a test OR when the admin triggers generation, call the Anthropic Claude API with this system:
You are an expert trading educator specializing in Smart Money Concepts for Indian equity markets (Nifty 50 and Bank Nifty). Generate advanced test questions that challenge traders beyond surface-level concept knowledge.

Generate {count} questions for Module: {module_title}
Difficulty: {difficulty}
Scenario Type: {scenario_type} — one of: [chart_reading, calculation, decision_making, concept_application, error_identification, multi_step_reasoning]

Requirements for each question:
1. Base it on a REALISTIC Nifty or BankNifty scenario with specific price levels (e.g., "BankNifty is at 48,240, PDH was 48,650, there's a bearish OB at 48,580–48,620...")
2. 4 options — only one correct, others plausible and commonly confused by traders
3. The wrong options must represent actual mistakes retail traders make
4. Explanation must be brutally honest and reference the exact concept
5. For 'expert' difficulty: require multi-concept reasoning (e.g., OB + FVG + premium/discount + MTFA all in one scenario)

Return ONLY valid JSON array:
[{
  "question_text": "...",
  "options": ["A", "B", "C", "D"],
  "correct_index": 0,
  "explanation": "...",
  "difficulty": "advanced",
  "scenario_type": "chart_reading",
  "is_ai_generated": true
}]
AI Generation Triggers:

On user test start — generate 3 fresh questions to supplement the 3 seeded questions (total 6-question test, need 4/6 to pass)
Daily cron job — generate 5 new questions per module per difficulty level and store in DB
Admin panel — manual generation with custom difficulty/scenario_type
